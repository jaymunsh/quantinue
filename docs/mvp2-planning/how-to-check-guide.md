## 1. 5분 확인 루틴 (매일)

### ① 앱이 살아 있나 — 10초

```bash
curl -s http://127.0.0.1:8020/health
# 기대: {"status":"ok","broker_mode":"mock","llm_mode":"local"}
```

- `llm_mode`가 `local`이어야 실 LLM 판단이다. `mock`이면 관측 무효.
- 안 뜨면: runbook §1대로 DB(5445) → 앱 순서로 켠다.

### ② 오늘 잡이 다 돌았나 — 1분

**텔레그램부터 본다.** 평일 KST 13:20 전후에 이런 한 통이 와야 한다:

```
✅ 2026-07-21 슬롯 · 잡 12/12 성공 · 신규 매수 3건
```

- **안 왔으면 그게 신호다** — 앱이 안 떴거나 맥이 꺼져 있었다.
- ❌로 시작하는 실패 알림이 왔으면 관제실에서 그 잡을 본다.
- 주말·공휴일엔 원래 안 온다. 정상.

직접 볼 때는 **http://127.0.0.1:8020/admin** (admin / quantinue-admin):

1. 잡 체인 12개가 전부 `succeeded`인가 — 아니면 화면이 끊긴 잡을 지목한다
2. 슬롯 탭에 빠진 날이 있는가
3. 계좌 총람에서 체결이 늘고 있는가

### ③ 유저 화면이 정직한가 — 1분 (가끔)

**http://127.0.0.1:8020/me** (user1 / quantinue-user)로 들어가서:

- 총자산 = 현금 + 보유 합계가 맞아 보이는가 (원장 대조 테스트가 이미
  강제하지만, 눈으로도 한 번)
- 체결마다 판단·근거가 붙어 있는가 (기계적 청산은 "규칙이 팔았다"로 표시)

## 3. "무인 완주" 판정 기준

조건 3이 채워지는 순간은 이렇다:

- **하루**: 사람이 아무것도 안 눌렀는데 텔레그램 ✅ 한 통이 왔다 = 그날 완주.
- **완성 선언**: 그런 날이 **며칠 연속** 쌓이고, 그 사이 실패 알림이 왔어도
  사람 개입 없이 다음 날 정상 복귀했으면 된다. (실패 0이 조건이 아니라
  **사람 개입 0**이 조건이다.)

관측 중 지켜야 할 것 둘 (runbook §3·§5):

- 관측 인스턴스(8020)를 **함부로 재시작하지 않는다** — 잡 도중 끄면 슬롯이
  `running`으로 굳는다. 굳었으면 관제실 `잠금 해제` 버튼.
- 코드 작업은 **8021로 따로 띄운다.** `--reload`는 작업용에만.

## 4. 완성 조건 밖에 남은 개발 (헷갈리지 말 것)

아래는 남아 있지만 **완성 조건이 아니다** (`completion-plan.md` 남은 것 표):

| 항목 | 성격 |
|---|---|
| 방어선 발동 내역 (W3-4 잔여) | 읽기 화면, 가벼움 |
| ⑦ SPY 벤치마크 잡 + /me 대비 | 잡 1종 + 화면 |
| ⑧ risk_rebuttal 등 프롬프트 계약 | 소비자 0 컬럼 채우기 |
| ⑨ LLM 예산 배선 | **AWS/openai 전환 시 선행 필수** |
| ⑩ pipeline_runs 스키마 정리 | 관측 끝나면 |

즉 **"로컬 기준 완성"은 관측 일수가 채워지면 선언할 수 있고**, 위 표는
그 뒤(또는 병행)의 개선 작업이다.
